在 Windows 命令行和 PowerShell 中显示非 ASCII 字符

在 Windows 环境下，只要使用的足够多，相信大家一定会遇到在命令行 (cmd.exe) 和 PowerShell 中显示非 ASCII 字符（比如中日韩字符，俗称 CJK 编码）的情况。但是有时候，命令行并补能够正常所有字符，取而代之的是方块 □ 或者问号 �。

为什么会出现这种情况？简单来讲：

It does not make sense to have a string without knowing what encoding it uses.
一个字符串如果不知道它用的编码，将毫无意义。
Joel Spolsky

换言之，命令行之所以出现乱码，都是因为编码不对。Windows 命令行的编码，默认都是 UCS-2（即 UTF-16，以前是 Java 和 JavaScript 中默认编码），这在二十年前，是相当常见的编码，基本上不会存在什么问题。但随着互联网的流行，UTF-8 编码变得越来越流行，使用 UTF-8 编码存储的文件也越来越普遍，这就有冲突了。如果感兴趣，可以看看本篇的 参考链接。
